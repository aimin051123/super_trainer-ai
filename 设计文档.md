## 1. 设计目标

### 1.1 功能目标

| 目标 | 说明 |
|------|------|
| 资料管理 | 支持多格式资料上传、自动分类归档 |
| 知识构建 | 将碎片知识编译为结构化知识图谱 |
| 精准练习 | 基于薄弱点生成针对性题目 |
| 复习规划 | 根据掌握度自动生成学习计划 |

### 1.2 质量目标

| 目标 | 指标 |
|------|------|
| 易用性 | 三个标签页完成所有操作，零配置启动 |
| 可理解性 | 代码量控制在 200 行以内，函数命名清晰 |
| 可追溯性 | 所有出题和回答标注来源 |
| 可扩展性 | 支持随时添加新学科、新资料 |

### 2.2 架构原则

| 原则 | 说明 |
|------|------|
| **函数即模块** | 每个智能体是一个独立函数，职责单一 |
| **文件即数据库** | 知识库用 Markdown 文件存储，无需数据库 |
| **统一调用** | 所有智能体共用一个 AI 调用函数 |
| **数据透传** | 模块之间通过参数传递数据，不共享状态 |

## 4. 智能体协作设计

### 4.1 智能体列表与职责

| 智能体 | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **资料解析** | 学科分类、实体识别 | 资料切片 | `{subject, type, summary}` |
| **知识编译** | 编织知识图谱、更新索引 | 原始内容+学科 | `{action, page, summary}` |
| **问答检索** | 基于知识库回答问题 | 问题+知识 | `str` (答案+来源) |
| **出题考官** | 出题、批改、更新掌握度 | 知识+难度+薄弱点 | `{questions: [...]}` |
| **复习规划** | 生成动态复习计划 | 薄弱点+日期+全部知识点 | `str` (计划表) |

### 4.2 协作流程

```
用户输入
    ↓
意图路由
    ├─ 上传资料 → 资料解析 → 知识编译 → 更新索引
    ├─ 提问 → 问答检索 → 回答 + 归档
    ├─ 出题 → 出题考官 → 批改 → 更新掌握度
    └─ 规划 → 复习规划 → 展示计划
```

### 4.3 协作时序图

```
用户         路由          资料解析       知识编译         知识库
 │             │             │             │              │
 │──上传资料──→│             │             │              │
 │             │──→解析─────→│             │              │
 │             │             │──分类结果──→│              │
 │             │             │             │──写入───────→│
 │             │             │             │←──完成───────│
 │             │             │             │              │
 │             │──→编译─────→│             │              │
 │             │             │             │──写入───────→│
 │             │             │             │←──完成───────│
 │             │             │             │              │
 │←──已收录─── │             │             │              │
```

### 4.4 智能体实现方式

每个智能体采用 **"提示词模板 + AI 调用 + 结果解析"** 模式：

```python
def parser(content):
    # 1. 构建提示词
    prompt = f"判断以下内容属于哪个学科：{content}"
    
    # 2. 调用 AI
    result = call_ai(prompt)
    
    # 3. 解析结果
    return {"subject": result}
```

| 步骤 | 说明 | 关键点 |
|------|------|--------|
| 构建提示词 | 告诉 AI 角色、任务、输出格式 | 提示词决定输出质量 |
| 调用 AI | 统一的 `call_ai()` 接口 | 所有模块共用 |
| 解析结果 | 转为程序可用的格式 | JSON 或纯文本 |

## 7. 流程设计

### 7.1 资料入库流程

```
用户上传文件
    ↓
提取文本（格式适配）
    ↓
智能切片（结构适配）
    ↓
┌──────────────────────────────┐
│ 对每个切片：                  │
│   Parser → 分类              │
│   写入 raw/{subject}/        │
│   Compiler → 编译            │
│   更新 compiled/{subject}/   │
│   更新 index.md              │
│   追加 wiki_log.md           │
└──────────────────────────────┘
    ↓
返回：收录结果
```

### 7.2 出题流程

```
用户请求出题
    ↓
读取掌握度缓存 
    ↓
掌握度缓存存在 → → → → → → → → → → → → → →  → → → → → → → → → → → → → → → → → → → → → → →掌握度缓存不存在
    ↓                                                                                         ↓ 
正常测试模式                                                                              摸底测试模式
     ↓                                                                                        ↓ 
获取薄弱点列表                                                                            读取知识页面内容
    ↓                                                                                         ↓ 
选择知识点：70% 薄弱 + 30% 已掌握                                                           根据全部章节知识点
    ↓                                                                                         ↓
读取知识页面内容                                                                               ↓
    ↓                                                                                         ↓
根据难度构建 Prompt ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← 
    ↓
调用 Examiner
    ↓
展示题目
    ↓
用户作答 → 批改
    ↓
更新掌握度
    ↓
返回：正确率 + 薄弱点更新
```

### 7.3 薄弱点判定流程

```
用户提交答案
    ↓
记录到 quiz_history
    ↓
查询该知识点全部记录
    ↓
计算：正确率、总次数、连续错误、最近表现
    ↓
判断：
  正确率<60% 且 次数≥3 → 薄弱
  连续错误≥2 → 薄弱
  近期正确率<40% 且 总正确率≥60% → 薄弱
  首次错误 且 次数<3 → 待观察
  否则 → 正常
    ↓
更新 mastery_cache
```

## 9. 部署设计

### 9.1 运行环境

| 项目 | 要求 |
|------|------|
| Python 版本 | 3.10+ |
| 操作系统 | Windows / macOS / Linux |
| 内存 | 最低 512MB |
| 存储 | 最低 100MB（不含资料） |
| 网络 | 需访问 DeepSeek API |

### 9.2 部署步骤

```bash
# 1. 安装 Python
# 2. 克隆项目
git clone https://github.com/yourname/SuperTutor.git
cd SuperTutor

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置 API Key
# 打开 app.py，替换 API_KEY = "sk-你的密钥"

# 5. 启动
streamlit run app.py
```

### 9.3 配置文件

```txt
# requirements.txt
streamlit>=1.28.0
openai>=1.3.0
requests>=2.31.0
pypdf>=3.17.0
python-pptx>=0.6.21
python-docx>=1.1.0
pydantic>=2.0.0
```

### 9.4 启动脚本

Windows (`start.bat`)：

```bat
@echo off
cd /d "%~dp0"
streamlit run app.py
pause
```

Mac/Linux (`start.sh`)：

```bash
#!/bin/bash
cd "$(dirname "$0")"
streamlit run app.py
```


## 10. 设计决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 开发语言 | Python | AI 生态丰富，零基础友好 |
| Web 框架 | Streamlit | 无需前端知识，200 行完成界面 |
| 知识存储 | 文件系统 (Markdown) | 透明、可读、无数据库依赖 |
| 模块实现 | 函数式（非类） | 零基础友好，代码更直观 |
| AI 调用 | 统一接口 | 便于维护和切换模型 |
| 状态管理 | st.session_state | Streamlit 原生支持 |
| 掌握度存储 | SQLite | Python 内置，无需额外安装 |
| 图片处理 | 提示用户补充 | 简化实现，避免引入 OCR |
