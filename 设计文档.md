## 第零章：功能速览

### 0.1 一句话定义

SuperTutor 是一个基于 DeepSeek API 的期末复习网页工具。上传课件 → AI 拆知识点 → 刷题 → 自动标薄弱 → 排复习计划。

### 0.2 七个功能

| 功能 | 做什么 |
|---|---|
| 知识收割 | 上传 PDF/PPT/Word/TXT/MD → 提取文字 → 按章节切片 → AI 判学科 → 存入知识库 |
| 碎片收割 | 手动输入笔记 → AI 判学科和类型 → 和上传资料存在同一个知识库 |
| 知识编译 | 原始碎片 → AI 重写为格式统一的 Markdown 页面 → 自动建双向链接 |
| 知识问答 | 检索知识库 Top-5 → 注入检索结果 + 历史对话 → SSE 流式回答 → 标注来源 |
| 精准刷题 | 选学科/难度/题量 → AI 基于知识点出题（外键绑定）→ 提交后本地批改 → 自动记录错题 |
| 薄弱点判断 | 4 条规则 if-else 自动判定 → 下次出题优先出薄弱点 |
| 复习规划 | 考试日期 + 掌握度 + 艾宾浩斯遗忘曲线 → AI 生成每日计划 |

### 0.3 学习闭环

```
上传资料 → AI提取知识点 → 编译为知识库
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
           知识问答        精准刷题         复习计划
           (SSE流式)     (外键绑定知识点)  (艾宾浩斯曲线)
                              │               │
                              ▼               │
                    薄弱点判断(4条规则)←──────┘
                              │
                    下次出题优先出薄弱点 ──→ 循环
```

### 1.2 模块职责与不可替代性

#### UI 层：Streamlit

**为什么不能用 Flask/Django**：需手写前端 JS 做 SSE 流式显示和状态管理，维护从 1 个 Python 文件变成 3 个文件。

#### 知识收割模块（parse_file / slice_text / classify_subject / harvest_file / harvest_fragment / extract_knowledge_points）

**为什么不能被"让用户贴纯文本"替代**：学生 90% 的资料是 PDF/PPT。

**学科分类的两级回退**：AI（`classify_subject` L344，temperature=0.3，json_mode=True）→ 失败则关键词词典匹配（L356-358）。

#### 知识点提取模块（extract_knowledge_points）

**为什么不能被"页面标题"替代**：页面改名历史数据全部断裂；知识点散落多页面时掌握度无法汇总。

**实现**：上传后 AI 返回 `{"knowledge_points":[{"name":"...","description":"...","importance":"high/medium/low"}]}` → `kp_upsert()` 写入 `knowledge_point` 表，掌握度初始 0.0。

#### 知识编译模块（compile_knowledge）

**为什么不能被"直接用原始切片"替代**：碎片格式混乱，AI 消化不了。

**双向链接实现**：第一遍编译收集 `linked_pages`（L488-489），第二遍为每个被引用页面追加"## 被以下页面引用"（L492-501）。

#### 精准刷题模块（generate_quiz / judge_weak / kp_update_mastery）

**出题 prompt 绑知识点 ID**（L633-642）：AI 返回 `knowledge_point_id` 字段，程序验证 ID 是否真实存在（L943-945），不存在则按 `source_page` 字符串匹配兜底（L947-952）。

**批改不调 AI**：用户选择首字母 == 正确答案首字母（L936），纯字符串比对。

#### 错题管理模块（wrong_record / wrong_list_unmastered / wrong_review）

**错题生命周期**：答题错误 → `wrong_record` 写入（L973-977）→ 错题本展示 → 用户重新作答 → 答对则 `mastered=1`（L1045-1051），答错则 `review_count++`（L222-228）。

**为什么不能靠 quiz_history 过滤**：错题和正确题混表，复习后仍躺在列表里。独立 `wrong_answer` 表支持复习次数追踪和掌握标记。

#### 复习规划模块（generate_plan）

**艾宾浩斯注入**（L688-694）：prompt 明确要求"第 1/2/4/7/15 天后安排复习"。

**为什么需要遗忘曲线**：不加间隔参数时 AI 会把同类内容堆在一起排列。

#### AI 调用层：call_ai()

**统一入口**（L243-264）：`prompt` / `system_prompt` / `json_mode` / `stream` / `temperature` 五个参数覆盖所有场景。

**为什么不能分散到各处**：换 API 提供商要改十几处；超时策略和错误格式不统一。

**SSE 批量刷新**（L267-282）：每 3 token 或遇标点才更新 UI，减少约 70% 重渲染。

**API Key 三级回退**（L23-34）：侧边栏输入 → 环境变量 `DEEPSEEK_API_KEY` → `config.properties`。

### 2.4 功能-数据映射

## 第三部分：核心流程

### 3.1 上传资料 → 知识点提取

```
parse_file() → classify_subject() → slice_text() → save_raw()（切片存 .md）
                                                  → extract_knowledge_points()（AI 提取知识点 → kp_upsert 入库）
```

两个操作并行：切片的 save_raw 不需要 AI，知识提取不需要切片（读全文前 4000 字）。

### 3.2 出题与批改（关键防错设计）

```
generate_quiz()
  → kp_list(cid) 查知识点列表
  → 构建 prompt（含 knowledge_point_id 外键要求）
  → AI 返回 JSON
  → 验证每道题的 knowledge_point_id 是否真实存在（L943-945）
  → 不存在则按 source_page 字符串匹配兜底（L947-952）

提交答案
  → quiz_record INSERT + last_insert_rowid() 在同一连接中（L922-928）
  → quiz_question INSERT（已验证的 kp_id）
  → kp_update_mastery(kp_id, ok) 更新掌握度
  → 答错 → wrong_record 写入错题表
```

**关键发现**：`db_query()` 每次新建连接，`last_insert_rowid()` 是连接级别的，必须在同一连接中执行 INSERT + SELECT（L922-928）。这是实测后发现并修复的 bug。

### 3.3 错题复习闭环

```
wrong_list_unmastered(cid) → 按知识点分组展示
  → 点击"重新作答" → 进入 review_mode
  → 隐藏正确答案，用户重选
  → 答对 → wrong_review(wid, ans, True) → mastered=1 → 更新知识点掌握度
  → 答错 → wrong_review(wid, ans, False) → review_count++ → mastered 保持 0
```

### 3.4 知识问答（多轮对话 + SSE 流式）

```
answer_question(question, course_id)
  → search_knowledge(question) 关键词检索 Top-5
  → chat_recent(course_id, 10) 取最近 10 条对话历史
  → 构建 system_prompt = 参考资料 + 历史对话
  → call_ai(stream=True) → stream_display() 批量刷新显示
  → chat_save() 保存本轮对话
```

### 4.2 错误回退策略

| 场景 | 主方案 | 回退 |

## 第六部分：调试经验与已知陷阱

### 6.1 `knowledge_point` 表索引（高频 bug 来源）

```
[0]=id  [1]=course_id  [2]=name  [3]=description  [4]=importance
[5]=mastery  [6]=total_count  [7]=correct_count  [8]=is_weak
[9]=weak_reason  [10]=last_reviewed
```

**常见错误**：把 `row[4]`（importance，TEXT）当成 mastery，把 `row[7]`（correct_count，INTEGER）当成 is_weak。正确是 `row[5]` 取 mastery，`row[8]` 取 is_weak。

### 6.2 `course` 表索引

```
[0]=id  [1]=name  [2]=description  [3]=exam_date
[4]=daily_study_minutes  [5]=created_at
```

**常见错误**：把 `course[4]`（daily_study_minutes，INTEGER）当成 exam_date，导致 `strptime()` 报错。正确是 `course[3]` 取考试日期。

### 6.3 SQLite `last_insert_rowid()` 陷阱

`last_insert_rowid()` 是**连接级别**的函数。`db_query()` 每次调用都新建并关闭连接，所以 INSERT 和 `SELECT last_insert_rowid()` 如果在两次 `db_query()` 调用中，后一次拿到的永远是 0 或 NULL。解决：INSERT 和 SELECT 必须在**同一个连接**中（L922-928）。

### 6.4 AI 返回值不可信

AI 可能返回不存在的 `knowledge_point_id`（幻觉）。必须验证：`if kp_id not in valid_kp_ids: kp_id = None`（L943-945）。

### 6.5 `except: continue` 静默吞错

`compile_knowledge()` 的 L479-480 在 AI 返回非 JSON 时 `continue` 静默跳过，同时 `results` 为空时返回"✅ 所有资料已是最新"。用户看到成功提示但实际什么都没编译。排查方法：直接检查 `知识库/编译后知识/` 文件夹是否为空。
