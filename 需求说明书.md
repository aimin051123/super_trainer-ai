## 一、项目背景

### 1.1 用户痛点

大学生在期末复习和考研备考中普遍面临四个问题：

| 痛点 | 具体表现 | SuperTutor 解决方案 |
|---|---|---|
| 资料分散，缺乏统一管理 | PDF 课件、PPT 讲义、Word 笔记、手机备忘录散落在不同设备 | 一键上传所有格式 → AI 自动分类归档、提取知识点 |
| 不知道自己哪里薄弱 | "感觉数据结构不太会"——但具体是二叉树不会还是图论不会？没有数据支撑 | 刷题数据驱动——每次答题自动更新掌握度，4 条规则精准判定薄弱点 |
| 复习缺乏计划性 | 离考试还有 20 天，每天能学 3 小时，但不知道怎么分配 | AI 结合艾宾浩斯遗忘曲线 + 各科考试日期 + 掌握度分布，生成每日计划 |
| 通用 AI 无法基于自有资料 | 直接问 ChatGPT 答案可能和教材不一致，甚至编造内容 | 检索你的知识库 → 注入 prompt → AI 只基于你的资料回答 → 标注来源 |

### 1.2 核心闭环

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 上传资料  │ →  │ AI提取   │ →  │ 编译为   │ →  │ 刷题练习  │
│ PDF/PPTX │    │ 知识点   │    │ 知识库   │    │ 四种模式  │
└──────────┘    └──────────┘    └────┬─────┘    └────┬─────┘
                                     │               │
                      ┌──────────────┼───────────────┤
                      ▼              ▼               ▼
                 知识问答        薄弱点判断        复习计划
              (基于自有资料)   (4条规则自动)    (艾宾浩斯曲线)
                                     │               │
                                     └─── 联动 ──────┘
                              薄弱点影响下次出题偏好
                              答题数据驱动计划动态调整
```

## 二、功能需求

### F1 知识收割 — 文件上传

#### 1.1 支持的文件格式

| 格式 | 解析库 | 提取方式 |
|---|---|---|
| PDF | `pypdf` | `PdfReader.pages[].extract_text()` |
| PPT/PPTX | `python-pptx` | 遍历 `slides[].shapes` 取 `text_frame.text` |
| DOC/DOCX | `python-docx` | 遍历 `paragraphs[]` 取 `text` |
| TXT/MD | Python 原生 | `read().decode("utf-8")`，失败试 GBK |
| JPG/PNG 等图片 | — | 提示"当前版本不支持图片，请使用电子文档" |

#### 1.2 上传处理流程

```
用户选择文件 → 拖入上传区域
  ↓
parse_file() 提取全部文字（二进制 → 字符串）
  ↓
classify_subject() 判断学科
  ├── 主方案：AI 读取前 3000 字 → 返回 {"subject": "数学"}
  └── 回退：关键词词典匹配（KW_DICT），全不中 → "其他学科"
  ↓
slice_text() 按标题切片
  ├── 匹配 #{1,4} 标题 → Markdown 格式
  ├── 匹配 1. / 1、/ 第一章 → 数字/中文序号
  └── 全不匹配 → 降级按空行分段
  ↓
save_raw() 逐片存储
  → 知识库/原始资料/{学科}/时间戳_标题.md
  → 顶部写入 frontmatter（type/subject/source/created）
  ↓
extract_knowledge_points() AI 提取知识点
  → 读全文前 4000 字 → AI 返回 [{name, description, importance}]
  → kp_upsert() 写入 knowledge_point 表（mastery=0.0）
```

#### 1.3 学科分类回退机制

```
AI 调用（temperature=0.3, json_mode=True）
  ├── 成功 → 返回学科名，自动创建对应文件夹
  └── 失败 → 关键词词典兜底
       ├── 默认学科：数据结构/操作系统/计算机网络/计算机组成原理
       ├── 扩展学科：数学（函数/映射/极限/导数/矩阵/向量/概率/数列/定理/证明/集合）
       ├──           英语（vocabulary/grammar/reading/writing/translation）
       ├──           物理（力学/电磁/光学/热学/量子/速度/加速度/能量/电场/磁场）
       └── 全不匹配 → "其他学科"
```

### F2 知识收割 — 碎片输入

- 用户在文本框手动输入笔记/易错点/顿悟
- `classify_entity()` 判断实体类型：概念 / 算法 / 对比 / 考试技巧
- `classify_subject()` 判断所属学科
- 与上传文件统一存入 `知识库/原始资料/{学科}/`
- 同时触发知识点提取

### F3 知识编译

#### 3.1 编译目的

原始资料是碎片化的：同一份 PDF 切出的多个片段、口语化的笔记、干巴巴的教材摘录、不同格式的 PPT 文字——格式混乱、内容可能重复。编译将它们统一为结构化、互相链接的知识页面。

#### 3.2 编译流程

```
compile_knowledge()
  ├── 扫描原始资料 list_raw()
  ├── 读已编译页面的 frontmatter.source 字段 → 过滤已处理的
  ├── 对每个新原始文件：
  │     ├── 调 AI（prompt 含已有页面列表 + 去重规则）
  │     ├── AI 返回 {title, subject, page_type, is_new, merge_with, content, linked_pages}
  │     ├── is_new=false 且 merge_with 非空 → 合并到已有页面（追加内容 + 更新 updated）
  │     └── is_new=true → 创建新页面
  ├── 双向链接回写：
  │     └── 遍历 linked_pages → 在目标页面末尾追加"## 被以下页面引用"
  ├── 更新全局索引.md
  └── 追加变更日志.md
```

#### 3.3 去重规则

AI prompt 中明确要求：对比已有页面，内容重叠 >70% 则 `is_new=false`，`merge_with` 填已有页面名。同一知识点的不同角度合并到已有页面，不新建。

### F4 知识问答

- 用户输入问题 → `search_knowledge()` 从编译后知识库检索
- 检索算法：遍历所有 .md 文件，数字符命中次数，标题命中额外 +10 分，取 Top-5
- 注入 system prompt：`"你是学习助手。请基于以下资料回答。如资料不足请诚实告知，不要编造。"` + 检索内容 + 最近 10 条对话历史
- SSE 流式输出：`stream_display()` 每 3 token 或遇中文标点刷新 UI
- 答案末尾标注来源：`📎 [[页面名]]（学科）`
- 对话保存到 `chat_history` 表，下次提问自动注入

### F5 精准刷题

#### 5.1 四种练习模式

| 模式 | 出题逻辑 | 学科选择 | 代码参数 |
|---|---|---|---|
| 薄弱点优先 | prompt 注入 `is_weak=1` 的知识点名称 | 单学科 | `weak_first=True` |
| 全面练习 | 所有知识点随机抽取 | 单学科 | `weak_first=False` |
| 专项突破 | 用户多选目标知识点 | 单学科 | `focus_kp_ids=[...]` |
| 模拟考试 | 跨所有有资料的学科综合出题 | 全学科 | `subject=None` |

#### 5.2 答题流程

```
选择学科/模式/题量 → 点击"生成题目"
  ↓
generate_quiz()
  ├── 优先读编译后知识（空则回退原始资料）
  ├── 查知识点列表（含掌握度和薄弱标记）
  ├── 拼 prompt："基于以下知识点列表生成 N 道选择题"
  │     ├── 知识点 ID + 名称 + 掌握度（薄弱 ⚠️ 标注）
  │     ├── 薄弱点优先时："优先针对以下薄弱知识点出题：进程调度, TCP握手"
  │     └── 专项突破时：只传选中知识点
  └── AI 返回 JSON：[{knowledge_point_id, question, options, answer, explanation, source_page, difficulty}]
  ↓
逐题展示
  ├── 当前题号 / 总题数 + 进度条
  ├── 薄弱点题目标注 [`薄弱点`] 标签
  └── radio 选项 → 点击"提交答案"
  ↓
本地批改（不调 AI）
  ├── ul[0] == cl[0]：用户选择首字母 vs AI 答案首字母
  ├── 显示对错 + 解析 + 来源页面
  └── 薄弱点题目显示 💡 提示
  ↓
全部完成 → "完成测验"
  ├── 创建 quiz_record（测验快照）
  ├── 逐题 INSERT quiz_question（关联 knowledge_point_id）
  ├── kp_update_mastery(kp_id, is_correct) 更新掌握度
  │     └── 重算 mastery → judge_weak() → UPDATE knowledge_point
  └── 答错 → INSERT wrong_answer
```

#### 5.3 knowledge_point_id 幻觉防护

AI 可能返回不存在的 ID（幻觉）。代码做两重验证：
1. `kp_id in valid_kp_ids` → 不通过则置 None
2. 仍为空 → 按 `source_page` 字符串匹配知识点名称
3. 匹配不上 → `kp_upsert()` 自动创建新知识点

### F6 薄弱点判定

**4 条规则**（`quiz/weakness.py` → `judge_weak()`）：

| 优先级 | 规则 | 条件 | 判定结果 | 心理学依据 |
|---|---|---|---|---|
| 1 | 规则 2 | 连续错误 ≥2 次 | 🔴 薄弱 | 近因效应：当下正在发生的错误最紧急 |
| 2 | 规则 1 | 答题 ≥3 次，正确率 <60% | 🔴 薄弱 | 练习饱和：练多还错，说明不是粗心 |
| 3 | 规则 3 | 总正确率 ≥60% 但近 3 次 <40% | 🔴 薄弱 | 遗忘曲线：以前会现在忘 |
| 4 | 规则 4 | 答题 <3 次且首次错误 | 👀 待观察 | 数据不足：样本太少，不下结论 |

执行顺序：规则 4 → 2 → 1 → 3。命中即返回，后续不执行。

### F7 错题管理

- 答错自动调用 `wrong_record()` 写入 `wrong_answer` 表
- 错题本展示：`wrong_list_unmastered(cid)` 查询 `WHERE mastered=0`
- 复习模式：展开 → 隐藏正确答案 → 用户重新作答 → 确认
  - 答对 → `wrong_review(wid, ans, True)` → `mastered=1`，知识点掌握度 +1，从列表消失
  - 答错 → `wrong_review(wid, ans, False)` → `review_count++`，继续保留
- 每个错题关联 `knowledge_point_id`，可追踪是哪个知识点的错题

### F8 复习规划

#### 8.1 设置项

| 设置 | 存储位置 | 说明 |
|---|---|---|
| 每日学习时长 | `course.daily_study_minutes` | 通过 `course_update()` 持久化 |
| 各科考试日期 | `st.session_state.exam_dates` | 折叠面板，每学科一个日期选择器 |
| 自定义要求 | 不持久化 | 文本框选填，生成时拼入 prompt |

#### 8.2 计划生成流程

```
规划页 → 设置每日时长 + 各科日期 + 自定义要求
  ↓
点击"生成复习计划"
  ↓
generate_plan(cid, daily_minutes, custom_req, exam_dates)
  ├── 收集各科日期（含剩余天数）
  ├── 收集知识点掌握度（从 knowledge_point 表）
  ├── 收集薄弱点列表（is_weak=1）
  ├── 拼 prompt：
  │     ├── 规则1：艾宾浩斯遗忘曲线（第 1/2/4/7/15 天复习）
  │     ├── 规则2：薄弱点优先安排在前期
  │     ├── 规则3：每天不超过 daily_minutes 分钟
  │     ├── 规则4：每天不超过 2 个学科
  │     ├── 规则5：最后 3 天总复习 + 错题回顾
  │     ├── 规则6（可选）：用户自定义要求
  │     └── 各科考试日期表 + 统计信息
  └── AI 返回 Markdown 格式计划
  ↓
展示计划 + 本周日历视图 + 今日时间表
  ↓
自动填充 today_tasks（薄弱点前 4 个）→ 联动总览页打卡
```

### F9 课程管理

- 侧边栏 `st.selectbox` 下拉切换课程
- 每个课程独立的知识点/答题记录/错题/计划
- 新建课程：输入名称，自动创建数据库记录
- 删除课程：级联删除所有关联数据（知识点/答题/错题/对话）
- 首次使用自动创建"默认课程"

### F10 今日打卡任务联动

- 规划页生成计划时，自动从薄弱点提取前 4 个任务 → 存入 `st.session_state.today_tasks`
- 总览页读取 `today_tasks`：为空则回退到薄弱点列表
- 每个任务点击切换 ☐ ↔ ✅（划掉效果）
- 全部完成显示 🎉
- 刷新页面后消失（未持久化，鼓励每次生成计划后重新打卡）

### 3.2 四页面详细内容

**📊 总览**
- 4 个统计卡片（知识总数 / 已掌握 / 薄弱 / 待巩固）
- 各学科掌握度进度条（平均 mastery）
- 薄弱点列表（按正确率升序，前 2 个标"⚠️ 最高优先级"）
- 今日打卡任务（可点击切换完成状态）

**📚 知识库**
- 文件上传区（支持多文件）
- 碎片收割文本框
- 状态筛选下拉框（全部 / 已掌握 / 待巩固 / 薄弱）
- 搜索输入框（按知识点名称筛选）
- 知识点列表（图标 + 名称 + 状态 + 百分比 + 类型 + 答题次数/正确次数 + 薄弱原因）
- 知识问答（SSE 流式多轮对话）

**🎯 练习**
- 学科下拉框（有编译内容的标 ✅）
- 模式下拉框（薄弱点优先 / 全面练习 / 专项突破 / 模拟考试）
- 题量下拉框（3/5/8/10）
- 专项突破时显示知识点多选列表
- 逐题展示：题目 + 4 选项 + 进度条 + 提交按钮
- 对错显示 + 解析 + 薄弱点提示
- 完成测验后的成绩总结
- 错题本：展开 + 隐藏答案 + 重新作答 + 标记已掌握

**📅 规划**
- 每日学习时长数字输入
- 各科考试日期折叠面板（每学科日期选择器 + 剩余天数）
- 自定义要求文本框（选填）
- 生成复习计划按钮
- 计划展示 + 本周日历视图 + 今日时间表
- 掌握度总览（每个知识点一条进度条）
- 薄弱点与待观察列表

## 四、数据存储

### 4.1 SQLite 数据库（6 张表）

#### course — 课程

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 主键 |
| name | TEXT | 课程名 |
| description | TEXT | 描述 |
| exam_date | TEXT | 考试日期 (yyyy-MM-dd) |
| daily_study_minutes | INTEGER | 每日学习分钟数 |

#### knowledge_point — 知识点（核心表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 主键 |
| course_id | INTEGER FK | 所属课程 |
| name | TEXT | 知识点名称 |
| description | TEXT | AI 提取的描述 |
| importance | TEXT | high/medium/low |
| mastery | REAL | 掌握度 0~1 |
| total_count | INTEGER | 总答题次数 |
| correct_count | INTEGER | 正确次数 |
| is_weak | INTEGER | 是否薄弱 0/1 |
| weak_reason | TEXT | 薄弱原因 |

#### quiz_record — 测验快照

| 字段 | 说明 |
|---|---|
| id, course_id | 主键 + 课程外键 |
| quiz_type | 'periodic' |
| total_questions / correct_count / accuracy | 统计信息 |

#### quiz_question — 每道题明细

| 关键字段 | 说明 |
|---|---|
| knowledge_point_id | 知识点外键（可为 NULL，SET NULL 级联） |
| content / options / correct_answer / user_answer | 题目数据 |
| explanation / difficulty | 解析和难度 |

#### wrong_answer — 错题生命周期

| 关键字段 | 说明 |
|---|---|
| review_count | 复习次数 |
| mastered | 是否已掌握 0/1 |
| last_reviewed | 上次复习时间 |

#### chat_history — 对话记忆

| 字段 | 说明 |
|---|---|
| role | 'user' 或 'assistant' |
| content | 消息内容 |

│   ├── 计算机组成原理/

## 五、AI 调用规范

| 场景 | temperature | json_mode | stream | 回退策略 |
|---|---|---|---|---|
| 学科分类 | 0.3 | ✅ | — | KW_DICT 关键词匹配 |

| 知识编译 | 0.7 | ✅ | — | 跳过该文件继续下一个 |
| 出题 | 0.9 | ✅ | — | 显示"题目解析失败，请重试" |
| 问答 | 0.7 | — | ✅ | 显示"AI调用失败" |

**代理配置**：`httpx.Client(proxy="http://127.0.0.1:7897")`  
**SSE 流式优化**：每 3 token 或遇中文标点（，。！？\n）刷新 UI，减少约 70% 重渲染

## 六、非功能性需求

| 类别 | 要求 |
|---|---|
| 代码架构 | 纯函数式，单入口 `main.py`（~360 行）+ 8 个模块包 |

| 浏览器 | Chrome/Edge 最新版，最低分辨率 1280×720 |
