## 1. 项目概述

### 1.1 项目背景

学生在期末复习或备考过程中面临三大核心痛点：

| 痛点 | 具体表现 |
|------|---------|
| **资料管理混乱** | 教材、PPT、笔记、真题分散存放，复习时难以定位重点 |
| **缺乏针对性** | 无法精准识别薄弱环节，会的反复练，不会的永远跳过 |
| **计划随意** | 缺乏科学的排期和动态调整机制，复习效率低下 |

### 1.2 项目愿景

构建一款 **"自生长的智能私教系统"** ，形成 **"输入→编译→出题→反馈→规划"** 的完整学习闭环。学生只需上传资料、完成摸底测试、设定考试时间，系统自动完成知识库构建、薄弱点诊断、针对性出题和动态复习排期。

### 1.3 核心设计哲学

系统采用 **"文件夹即知识库"** 的设计理念，所有知识以 Markdown 形式存储在本地文件夹中。系统不依赖预训练知识，所有回答和出题均基于用户上传的资料。


## 2. 用户角色与使用场景

### 2.1 用户画像

| 属性 | 描述 |
|------|------|
| **身份** | 期末复习的大学生或备考研究生入学考试的学生 |
| **学习习惯** | 每天学习 4-6 小时 |
| **核心诉求** | 找到薄弱点、精准练习、科学规划 |

### 2.2 核心使用场景

| 场景 | 描述 |
|------|------|
| **资料收割** | 上传教材/PPT，或记录易错点和顿悟 |
| **摸底测试** | 首次使用时快速定位薄弱环节 |
| **精准刷题** | 基于薄弱点进行针对性练习，支持三级难度 |
| **查看计划** | 查看系统根据掌握度生成的复习计划 |


## 3. 系统架构

### 3.1 整体架构

系统采用三层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                     表示层（用户界面）                        │
│                  知识收割  │  刷题  │  规划                   │
└─────────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────────┐
│                     业务逻辑层（Agent 调度）                  │
│                                                             │
│    资料解析官 → 知识编译官 → 出题考官 → 复习规划官             │
│                     ↓                                       │
│                 统一 AI 调用接口                             │
└─────────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────────┐
│                     数据层（知识库）                          │
│                                                             │
│    文件夹（Markdown 知识库）  +  SQLite（掌握度缓存）          │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 知识库文件夹结构

```
/知识库/
├── 全局索引.md              # 所有页面的目录（自动维护）
├── 变更日志.md              # 每次修改的记录
├── 原始资料/                # 上传资料的原始切片（只读）
│   └── {学科名}/
│       └── 日期_摘要.md
├── 编译后知识/              # 结构化知识图谱（读写）
│   ├── 问答归档/            # 历史问答记录
│   └── {学科名}/
│       └── 知识点.md
└── 缓存/                    # 去重缓存
```



## 4. 功能性需求

### 4.1 模块一：知识收割

#### 4.1.1 资料上传

| 功能 | 说明 |
|------|------|
| 支持格式 | PDF、PPT、Word、TXT、Markdown |
| 上传方式 | 拖拽上传或点击选择，支持批量上传 |
| 学科指定 | 用户手动选择或系统自动识别 |
| 处理流程 | 上传 → 提取全文 → 智能切片 → 归档到原始资料 |

**三层适配架构**：

| 层级 | 功能 | 说明 |
|------|------|------|
| 格式适配层 | 提取纯文本 | 各格式统一转为纯文本 |
| 结构适配层 | 智能切片 | 按章节标题、段落语义切割 |
| 语义适配层 | 分类归档 | AI 判断学科、类型、摘要 |

**智能切片策略**（按优先级）：

| 优先级 | 策略 | 识别模式 |
|--------|------|----------|
| 1 | 中文章节 | `第X章`、`第X讲` |
| 2 | Markdown标题 | `##`、`###` |
| 3 | 英文章节 | `Chapter`、`Section` |
| 4 | 中文序号 | `一、`、`（一）` |
| 5 | 段落分割 | 双空行 |
| 兜底 | 固定字数 | 每2000字符一段 |

**图片处理策略**：

| 资料类型 | 处理方式 |
|----------|----------|
| 文字型资料 | 自动提取所有文字 |
| 含图片资料 | 提示用户补充文字描述 |
| 纯图片资料（扫描件） | 提示先OCR转换 |

#### 4.1.2 碎片知识收割

| 功能 | 说明 |
|------|------|
| 输入方式 | 聊天框输入文字片段 |
| 适用场景 | 错题分析、突然顿悟、重点总结 |
| 处理流程 | 识别学科 → 写入原始资料 |

#### 4.1.3 知识编译

| 功能 | 说明 |
|------|------|
| 触发时机 | 原始资料写入完成后自动触发 |
| 编译逻辑 | 判断属于已有页面或新建页面，更新编译后知识 |
| 冲突处理 | 如与已有知识冲突，用警告区块标出 |
| 双向链接 | 自动识别 `[[页面名]]` 并建立反向引用 |

#### 4.1.4 索引维护

| 功能 | 说明 |
|------|------|
| 全局索引 | 更新全局索引，新增条目并更新统计 |
| 变更日志 | 追加变更日志，记录每次变更 |

### 4.2 模块二：精准刷题

#### 4.2.1 摸底测试

| 功能 | 说明 |
|------|------|
| 触发 | 用户首次使用或手动点击 |
| 题量 | 5-8 道选择题 |
| 难度分布 | 基础30% + 中等50% + 困难20% |
| 输出 | 正确率 + 薄弱点列表 |

#### 4.2.2 三级难度控制

| 难度 | 适用场景 | 题目特征 |
|------|----------|----------|
| **基础** | 初次学习 | 概念辨析、简单记忆 |
| **中等** | 巩固练习 | 综合分析、比较判断 |
| **困难** | 冲刺提高 | 深度推理、陷阱识别 |

**各难度设计**：

| 设计维度 | 基础题 | 中等题 | 困难题 |
|----------|--------|--------|--------|
| 考核目标 | 概念记忆 | 理解应用 | 深度推理 |
| 题干复杂度 | 一句话 | 带情境 | 带条件和陷阱 |
| 干扰项特点 | 明显错误 | 有迷惑性 | 表面都合理 |
| 知识跨度 | 1个知识点 | 2-3个知识点 | 跨章节综合 |

**自动匹配规则**：

| 掌握度 | 分配难度 |
|--------|----------|
| 低（正确率<50%） | 基础为主 |
| 中（正确率50-80%） | 中等为主 |
| 高（正确率≥80%） | 困难为主 |

**用户手动控制**：支持手动选择"基础/中等/困难/自动匹配"。

#### 4.2.3 针对性练习

| 功能 | 说明 |
|------|------|
| 出题逻辑 | 70%来自薄弱点，30%来自已掌握知识点 |
| 题量 | 5道选择题 |
| 难度适配 | 薄弱点自动提升难度档位 |
| 数据源铁律 | 仅基于用户上传资料出题，严禁编造 |

#### 4.2.4 自动批改

| 功能 | 说明 |
|------|------|
| 批改 | 自动比对答案，计算正确率 |
| 薄弱标记 | 答错的知识点标记为薄弱 |
| 解析展示 | 每道题显示解析和来源页面 |

### 4.3 模块三：复习规划

| 功能 | 说明 |
|------|------|
| 计划生成 | 根据考试日期和薄弱点生成每日复习任务 |
| 动态调整 | 每次练习后重新计算掌握度，更新后续计划 |
| 进度可视化 | 展示各学科掌握度和每日任务清单 |


## 5. 薄弱点判断机制

### 5.1 判定维度

| 维度 | 说明 |
|------|------|
| **正确率** | 该知识点所有答题的正确比例 |
| **答题次数** | 该知识点被考察的总次数 |
| **最近表现** | 最近3次答题的正确情况 |
| **连续错误** | 连续答错的次数 |

### 5.2 薄弱判定规则

满足以下任一条件即判定为薄弱点：

| 规则 | 条件 |
|------|------|
| **低正确率** | 答题≥3次且正确率<60% |
| **连续错误** | 连续答错≥2次 |
| **近期下滑** | 总正确率≥60%但最近3次正确率<40% |
| **首次错误** | 答题<3次且首次答题错误 |

### 5.3 掌握度判定

| 状态 | 条件 |
|------|------|
| **已掌握** | 答题≥5次且正确率≥85% |
| **待巩固** | 答题≥3次且正确率60%-85% |
| **薄弱** | 满足上述任一薄弱规则 |

### 5.4 薄弱点生命周期

```
首次答错 → 待观察
    ↓
再次答错或正确率<60% → 薄弱点
    ↓
针对性练习（出题优先级最高）
    ↓
连续3次答对或正确率≥85% → 已掌握
```


## 6. Multi-Agent 设计

### 6.1 Agent 列表与职责

| Agent | 职责 |
|-------|------|
| **资料解析官** | 学科分类、实体识别、切片归档 |
| **知识编译官** | 编织知识图谱、更新索引和链接 |
| **问答检索官** | 基于知识库回答问题 |
| **出题考官** | 根据薄弱点和难度出题、批改 |
| **复习规划官** | 生成和动态调整复习计划 |

### 6.2 Agent 协作流程

```
用户输入
    ↓
意图路由
    ├─ 上传资料 → 资料解析官 → 知识编译官 → 更新索引
    ├─ 提问 → 问答检索官 → 回答 + 归档
    ├─ 出题 → 出题考官 → 批改 → 更新掌握度
    └─ 规划 → 复习规划官 → 展示计划
```

### 6.3 核心约束

| 约束 | 说明 |
|------|------|
| 数据源铁律 | 所有Agent基于用户上传资料，严禁编造 |
| 可追溯性 | 每道题和每个答案标注来源页面 |
| 冲突处理 | 知识冲突时用警告标出，不覆盖 |



## 7. 用户界面设计

系统采用三标签页布局，所有操作在浏览器中完成。

### 7.1 知识收割页

| 区域 | 功能 |
|------|------|
| 上传资料区 | 拖拽或点击上传，支持多格式 |
| 碎片收割区 | 输入易错点和顿悟 |
| 知识问答区 | 基于知识库回答问题 |
| 知识库总览 | 显示页面数、掌握度、薄弱点 |

### 7.2 精准刷题页

| 区域 | 功能 |
|------|------|
| 控制区 | 选择学科、难度、题量 |
| 题目区 | 展示题目和选项 |
| 批改区 | 提交答案后显示对错和解析 |
| 进度区 | 显示正确率和薄弱点更新 |

### 7.3 复习规划页

| 区域 | 功能 |
|------|------|
| 设置区 | 考试日期、每日学习时长 |
| 掌握度总览 | 各学科进度可视化 |
| 薄弱点列表 | 集中展示所有薄弱点 |
| 计划展示 | 每日复习任务清单 |
